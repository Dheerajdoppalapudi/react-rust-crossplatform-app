## slide_deck

Interactive presentation — generates a navigable, beautifully styled HTML slide deck rendered inside the lesson. The HTML is generated by a dedicated LLM call that produces a complete, self-contained presentation with navigation, progress bar, keyboard shortcuts, and download support.

**When to use:** Only when the user explicitly asks for a "presentation", "slides", "PPT", or "slide deck". Do NOT use by default in regular lessons.

**Props:**
- `spec` (string, **required**): plain-English description of the presentation content — include the title, target audience, number of slides, and key points per slide. Be detailed; this drives the slide generator.
- `title` (string, optional): presentation title shown in the React wrapper header

**How to write `spec`:**

Write a concise but complete description covering:
1. The presentation title
2. Target audience (e.g. "high school students", "software engineers")
3. Slide count (aim for 6–10)
4. For each slide: heading + 3–5 bullet points or a paragraph

Example spec:
```
Title: "Introduction to Machine Learning"
Audience: undergraduate CS students with no ML background
Slides: 8

Slide 1 (title): "Introduction to Machine Learning" — subtitle: "From Data to Decisions"
Slide 2 (bullets): "What is Machine Learning?" — machines learn patterns from data without explicit rules; three paradigms: supervised, unsupervised, reinforcement; used in search, recommendations, vision
Slide 3 (bullets): "Supervised Learning" — learns from labeled examples; predicts output for new input; examples: spam detection, house price prediction, image classification
Slide 4 (two-column): "Supervised vs Unsupervised" — left: Supervised: labeled data, known output, classification & regression | right: Unsupervised: unlabeled data, discover structure, clustering & dimensionality reduction
Slide 5 (bullets): "Key Algorithms" — Linear/Logistic Regression (simple, interpretable), Decision Trees & Random Forests (powerful, handles non-linearity), Neural Networks (state-of-the-art for complex tasks)
Slide 6 (content): "The ML Pipeline" — paragraph: Data collection and cleaning → Feature engineering → Model selection → Training → Evaluation → Deployment. Each step is critical; bad data produces bad models ("garbage in, garbage out").
Slide 7 (bullets): "Real-World Applications" — Computer vision: object detection, medical imaging; NLP: translation, chatbots, summarization; Recommendation systems: Netflix, Spotify, Amazon; Autonomous vehicles
Slide 8 (title): "Summary" — subtitle: "Supervised · Unsupervised · Reinforcement · Data is everything"
```

**Entity format:**
```json
{
  "id": "sd1",
  "type": "entity",
  "entity_type": "slide_deck",
  "props": {
    "title": "Introduction to Machine Learning",
    "spec": "Title: 'Introduction to Machine Learning'\nAudience: undergraduate CS students\nSlides: 8\n\nSlide 1 (title): ..."
  }
}
```

**Important notes:**
- The `spec` is the input to a slide-generator LLM — the more detail you provide, the better the output.
- Do NOT include actual HTML or JSON slide arrays in `spec` — describe the content in plain English.
- Only use this entity when the user explicitly requests a presentation, not in regular lessons.
