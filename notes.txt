Current process flow diagram
User Prompt
    ↓
create_plan() — 1 LLM call
    classifies intent_type, decides frame_count,
    writes frame descriptions + narration + captions
    ↓
Route by intent_type:
    ├── process / architecture / timeline
    │       → generate_mermaid_frames()
    │         LLM writes Mermaid text → Node sidecar → Excalidraw elements
    │
    ├── math
    │       → generate_manim_frames()
    │         LLM writes Python/Manim → subprocess renders → PNG
    │
    └── concept_analogy / comparison / illustration
            → generate_all_frames()
              LLM writes raw slim JSON (x, y, width, height coords) → enhance()
    ↓
combine_frames() — stitch frames side by side
    ↓
enhance() — slim JSON → full .excalidraw format
    ↓
Return .excalidraw + narration text per frame


Drawbacks of illustrations flow:
      LLMs are very bad at spatial reasoning in pixel coordinates. They don't "see" a canvas. They're pattern-matching from training data, not reasoning about space.
      The shape vocabulary is too primitive. Rectangles + ellipses + lines can't make a convincing robot doing a flip, let alone explain thermodynamics with a meaningful visual.
      There's no feedback loop. The LLM generates blind — it can't see the output and fix proportions.
      Thermodynamics-style concept diagrams (molecules, heat flow, pressure) are particularly bad here because they need curves, gradients, and accurate spatial metaphors that require real drawing tools.
